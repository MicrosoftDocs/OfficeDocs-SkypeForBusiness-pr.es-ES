# <a name="configure-networking"></a>Configurar la red

| No | Tarea o actividad | Descripción | ¿Completado? | Información adicional |
|----|--------------------------------------------------------------------|-----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|------------|------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| 1  | Revise los requisitos de red para equipos | Tener una visión general de los requisitos de red antes de entrar en los detalles de la red. | | [Preparar la red de la organización para Microsoft Teams](https://docs.microsoft.com/microsoftteams/prepare-network)                                                               |
| 2  | Entregar el taller de preparación de la red | Completar el taller de preparación de la red de MyAdvisor y trabajar con sus equipos para identificar áreas de preocupación para la dirección como parte del esfuerzo de corrección. | | [Activos de disponibilidad de red MyAdvisor](https://myadvisor.fasttrack.microsoft.com/CloudVoice/Downloads?SelectedIDs=4_3_0_2,4_3_0_3,4_3_0_5,4_3_0_6,4_3_0_7,4_3_0_8,4_3_0_10,4_3_0_11) |
| 3  | Utilice el planificador de la red | Utilice el planificador de la red para calcular el ancho de banda requerido para todas las ubicaciones y los sitios en el ámbito. | | [Planificador de red MyAdvisor](https://myadvisor.fasttrack.microsoft.com/CloudVoice/NetworkPlanner)                                                                                   |
| 4  | Validar el tamaño del grupo NAT necesario para la conectividad del usuario | Asegúrese de que las direcciones IP públicas adecuadas se asignan a las agrupaciones NAT para evitar el agotamiento de puerto. Agotamiento de puerto contribuirá a usuarios internos y los dispositivos que no puede conectarse al servicio de Office 365. <br/><br/>Problemas de conectividad son la principal causa de problemas de la percepción del usuario para los servicios de nube. | | [Compatibilidad con NAT con Office 365](https://support.office.com/article/NAT-support-with-Office-365-170e96ea-d65d-4e51-acac-1de56abe39b9) |
| 5  | Implementar el enrutamiento más eficaz para centros de datos de Microsoft | Identificar ubicaciones que pueden usar puntos de egreso locales o regionales para conectarse a la red de Microsoft de forma más eficaz posible. <br/><br/>El artículo en la columna **información adicional** describe cómo los clientes pueden aprovechar características de resolución de nombres de Office 365 y enrutamiento IP para conectarse eficientemente al centro de datos regional más cercano. | | [Conectividad de cliente de Office 365](https://support.office.com/article/Client-connectivity-4232abcf-4ae5-43aa-bfa1-9a078a99c78b) |
| 6  | Configurar puertos de firewall que solicitó para la conectividad a los equipos | Revise las direcciones URL de Office 365 e IPs para identificar y probar los puertos de firewall que se requieren para la conectividad entre clientes locales y servidores y servicios de Office 365. <br/><br/>Para un entorno compatible, debe abrir todos los puertos en los servidores de seguridad. Error al abrir algunos puertos o intervalos afectará negativamente la experiencia del usuario. Configurar puertos de medios en los servidores de seguridad que se basa en las instrucciones de la columna de **información adicional** . | | [Office 365 las direcciones URL y direcciones IP a equipos de Microsoft](https://support.office.com/article/office-365-urls-and-ip-address-ranges-8548a211-3fe7-47cb-abb1-355ea5aa88a2#bkmk_teams) |
| 7  | Configurar los servidores proxy | Configurar el entorno para que se omiten los servidores proxy y puede conectar los usuarios directamente a Office 365 mediante el uso de UDP, para el mejor audio y vídeo de calidad. Cuando los medios de comunicación en tiempo real se ve obligado a atravesar un servidor proxy, se fuerza la pila de multimedia en equipos no TCP, que afecta negativamente a la calidad. <br/><br/>Para la experiencia del usuario de mayor calidad, siempre prefiere UDP TCP. | | [Planificación para la gestión de servicio y calidad](https://docs.microsoft.com/MicrosoftTeams/envision-planning-for-service-management-and-quality-complete-guide) |
| 8  | Configurar VPN de túnel de división | Se recomienda que proporcione una ruta alternativa para el tráfico de los equipos que omite la VPN, comúnmente conocida como *división túnel VPN*. Túnel dividido significa que el tráfico para Office 365 no recorren la VPN pero va directamente a Office 365. Este cambio tendrá un impacto positivo sobre la calidad, pero también ofrece la ventaja de reducir la carga de los dispositivos VPN y la red de la organización secundaria. | | Para implementar una VPN de túnel dividido, consulte a su proveedor VPN para los detalles de configuración. |
| 9  | Configurar la prioridad de los paquetes mediante QoS | QoS debe implementarse en todos los segmentos de una red administrada. Incluso cuando una red se haya adecuadamente aprovisionada para ancho de banda, QoS proporciona la mitigación del riesgo en caso de eventos imprevistos. Cuando se implementa QoS, se priorice el tráfico de voz para que estos eventos no previstos no afectan negativamente la calidad. | | [Planificación para la gestión de servicio y calidad](https://docs.microsoft.com/MicrosoftTeams/envision-planning-for-service-management-and-quality-complete-guide) <br/><br/>[Calidad de servicio en equipos de Microsoft](https://docs.microsoft.com/MicrosoftTeams/qos-in-teams) |
| 10 | Optimizar las redes Wi-Fi de calidad y rendimiento | Varios factores entran en juego al optimizar su red Wi-Fi: <ul><li>Se priorice la implementación de QoS o Multimedia Wi-Fi (WMM) para garantizar que el tráfico multimedia a través de las redes Wi-Fi.</li><li>Planificación y optimización de las bandas de Wi-Fi y acceso punto de colocación. El intervalo de 2,4 GHz puede proporcionar un rendimiento adecuado, dependiendo de la posición del punto de acceso.</li></ul> Consulte a su proveedor de Wi-Fi para obtener instrucciones específicas. | | [Recomendaciones de Wi-Fi para extremos](https://docs.microsoft.com/MicrosoftTeams/envision-planning-for-service-management-and-quality-complete-guide#wi-fi-recommendations-for-endpoints) |
| 11 | Validar la conectividad de red mediante la herramienta de evaluación de la red | Utilizar la herramienta de evaluación de la red de Skype para empresas y equipos para probar la conectividad a todas las direcciones IP y puertos utilizados en Skype para reuniones y llamadas de negocios en línea y equipos. Descargue la herramienta y consulte Usage.docx para obtener más información acerca de cómo utilizar la herramienta e interpretar los resultados de la prueba. Se recomienda ejecutar la herramienta desde un PC cliente en cada ubicación donde se va a utilizar los equipos. | | [Skype para empresas y la herramienta de evaluación de red de los equipos](https://go.microsoft.com/fwlink/?linkid=855799) |
