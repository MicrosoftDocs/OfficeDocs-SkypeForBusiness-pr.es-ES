# <a name="configure-microsoft-teams-core-capabilities"></a>Configurar capacidades centrales de Microsoft Teams

| No | Tarea o actividad | Descripción | ¿Completado? | Información adicional |
|----|-----------------------------------------------------------------|--------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|------------|---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| 1  | Validar que su entorno incluye todos los requisitos previos de los equipos | Equipos depende de otras plataformas para construir una solución de colaboración end-to-end. Trabajar con sus equipos de TI para asegurarse de que ha implementado y configurado correctamente Exchange, SharePoint Online y OneDrive para el negocio. | | [Interacción de SharePoint Online y OneDrive para la Empresa con Microsoft Teams](https://docs.microsoft.com/MicrosoftTeams/sharepoint-onedrive-interact) <br/><br/>[Interacción entre Exchange y Microsoft Teams](https://docs.microsoft.com/MicrosoftTeams/exchange-teams-interact) |
| 2  | Confirme que los equipos está habilitado para el arrendatario | Equipos está activada de forma predeterminada en todas las organizaciones. Compruebe la página de **servicios y complementos** en el centro de administración de Office 365 para comprobar que los equipos está habilitado para los inquilinos y habilitarlo si es necesario. | | [Configurar Microsoft Teams en su organización de Office 365](https://docs.microsoft.com/MicrosoftTeams/office-365-set-up) |
| 3  | Configurar funciones y permisos | Los equipos admiten dos tipos de funciones: miembro y propietario. <br/><br/>Después de agregar a un miembro a un grupo, un propietario también puede promover a un miembro a la función de propietario. Como práctica recomendada, se recomienda que haya al menos dos propietarios asignados a cada equipo. <br/><br/>De forma predeterminada, todos los miembros de la organización que tenga un buzón alojado en Exchange Online pueden crear un equipo. El usuario que crea un nuevo equipo se concede automáticamente el rol de propietario de ese equipo. <br/><br/>Si se necesita, puede configurar la configuración de grupo de Office 365 para permitir sólo determinados usuarios crear nuevos equipos. | | [Asignar roles y permisos en Microsoft Teams](https://docs.microsoft.com/MicrosoftTeams/assign-roles-permissions) <br/><br/>[Grupos de Office 365 y Microsoft Teams](https://docs.microsoft.com/MicrosoftTeams/office-365-groups) <br/><br/>[Administrar quién puede crear grupos de Office 365](https://support.office.com/article/Manage-who-can-create-Office-365-Groups-4c46c8cb-17d0-44b5-9776-005fced8e618) |
| 4  | Configurar el inquilino equipos | Puede configurar algunas opciones de equipos en el nivel de inquilinos. Usuarios habilitados para equipos hereden esta configuración de la configuración de clientes:<ul><li>General</li><li>Integración del correo electrónico</li><li>Aplicaciones</li><li>Almacenamiento en nube personalizado</li><li>Llamadas y reuniones</li><li>Mensajes</li></ul>| | [Habilitar características de Microsoft Teams en su organización de Office 365](https://docs.microsoft.com/MicrosoftTeams/enable-features-office-365) |
| 5  | OPCIONAL: Configurar el acceso de invitado | Utilice el acceso como invitado en equipos para colaborar con usuarios externos a la organización y les concede acceso a los equipos y los canales. Acceso como invitado es una configuración de inquilinos en los equipos. Está desactivada de forma predeterminada. <br/>Habilitar el acceso como invitado y configurar todo el inquilino invitado, si su organización tiene previsto utilizar esa característica. | | [Acceso de invitado en Microsoft Teams](https://docs.microsoft.com/MicrosoftTeams/guest-access) |
| 6  | OPCIONAL: Configurar la directiva de nomenclatura de equipos | Equipos aprovecha las políticas de nomenclatura para los grupos de Office 365 cuando los usuarios crean o editan los nombres de equipo. <br/><br/>De forma predeterminada, no hay restricciones de nomenclatura se aplican cuando un usuario crea un equipo. <br/><br/>Si necesita aplicar las reglas para nombres de equipos, configurar Office 365 grupos nomenclatura de directivas que se aplican a su organización. Puede establecer obligatorio prefijos y sufijos y especificar palabras bloqueadas. | | [Planificar grupos de Office 365 al crear equipos en Microsoft Teams](https://docs.microsoft.com/microsoftteams/plan-office-365-groups) <br/><br/>[Directiva de nomenclatura de grupos de Office 365](https://support.office.com/article/Office-365-Groups-naming-policy-6ceca4d3-cad1-4532-9f0f-d469dfbbb552) |
| 7  | Configurar Exchange para el dominio SMTP de los equipos | Equipos utiliza Exchange Online para enviar notificaciones a los miembros del equipo mediante el dominio SMTP — email.teams.microsoft.com — cuando haya ha agregado o quitado. <br/><br/>No olvide agregar este dominio SMTP a la lista de dominios aceptados en la infraestructura de Exchange. | | [Agregar dominio SMTP de Microsoft Teams como un dominio aceptado en Exchange Online](https://docs.microsoft.com/MicrosoftTeams/smtp-accepted-domain) |
| 8  | Configurar y administrar el acceso de usuario a equipos | Aunque recomendamos que habilite a todos los usuarios de equipos, puede permitir o denegar el acceso a los equipos en cada usuario asignando o retirar la licencia del producto de los equipos. | | [Gestionar acceso de los usuarios a Microsoft Teams](https://docs.microsoft.com/MicrosoftTeams/user-access) |
| 9  | Asignar licencias a los usuarios | Asignar licencias a los usuarios características como conferencias de Audio, sistema de teléfono y llamar a planes | | [Asignar licencias de Skype Empresarial y Microsoft Teams](https://docs.microsoft.com/SkypeForBusiness/skype-for-business-and-microsoft-teams-add-on-licensing/assign-skype-for-business-and-microsoft-teams-licenses) <br/><br/>[MyAdvisor: las secuencias de comandos de habilitación de usuario](https://myadvisor.fasttrack.microsoft.com/CloudVoice/Downloads?SelectedIDs=5_2_0_6,5_2_0_3) |
| 10 | Opcional: Use PowerShell para administrar equipos | Puede usar cmdlets de PowerShell en lugar de con el centro de administración de Office 365 para administrar y gestionar la configuración de los equipos. | | [Equipos de Microsoft PowerShell](https://docs.microsoft.com/powershell/module/teams/?view=teams-ps) |
