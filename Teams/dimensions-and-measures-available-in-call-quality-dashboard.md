# <a name="dimensions-and-measures-available-in-call-quality-dashboard"></a>Dimensiones y medidas disponibles en el Panel de calidad de llamadas

El panel de calidad de llamadas (CQD) para Microsoft Teams y Skype para profesionales en línea permite obtener entendimiento de la calidad de las llamadas realizadas mediante Microsoft Teams y Skype para servicios de negocios. En este tema se ofrece información detallada sobre las dimensiones y las medidas que se muestran en este panel. Para obtener más información sobre CQD y cómo habilitarlo, vea [activar y con el panel de calidad de llamadas para los equipos de Microsoft y Skype para profesionales en línea](turning-on-and-using-call-quality-dashboard.md).



## <a name="first-and-second-endpoint-classification"></a>Clasificación de primer y segundo punto de conexión

Muchas de las dimensiones y medidas del panel de calidad de llamadas se etiquetan como primeras o segundas. La siguiente lógica determina qué punto de conexión implicado en la transmisión o la llamada se etiqueta como primero:

- En primer lugar siempre será un extremo de servidor (MCU AV, el servidor de mediación, etc.) si un servidor está involucrado en la llamada de secuencia.

- Como segundo punto de conexión siempre se encuentra un punto de conexión de cliente a menos que la transmisión se dé entre dos puntos de conexión de servidor.

- Si ambos extremos son del mismo tipo, el orden para el que es el primero frente en segundo lugar se basa en orden interno de la categoría de agente de usuario. De este modo se asegura que el orden es coherente.

### <a name="notes-on-dimension-data-typeunits"></a>Notas sobre el tipo de datos de dimensión o unidades:

#### <a name="range"></a>Intervalo

Las dimensiones que se proporcionan como rango o grupo de valores se muestran con el siguiente formato:

 _\<cadena de criterio de ordenación\> [\<límite inferior inclusivo\> - \<límite superior exclusivo\>)_

Por ejemplo, la dimensión de duración (minutos) representa la duración en segundos de la llamada con el valor que se notifiquen como un rango de valores.

|||
|:-----|:-----|
|**Duration (Minutes)** <br/> |**Cómo interpretar** <br/> |
|062: [0-0)  <br/> |Duración en secuencia = 0 minutos  <br/> |
|064: [1-2)  <br/> |1 minuto < = < de duración de la secuencia 2 minutos  <br/> |
|065: [2 - 3)  <br/> |2 minutos < = secuencia duración < 3 minutos  <br/> |
|066: [3 - 4)  <br/> |3 minutos < = secuencia duración < 4 minutos  <br/> |
|…  <br/> |…  <br/> |

El <sort order string> se usan para controlar el criterio de ordenación al presentar los datos y se puede usar para filtrar. Por ejemplo, un filtro de la duración (en minutos) < "065", mostraría secuencias con duración menor que 2 minutos (a la izquierda '0' es necesario para que el filtro para que funcione como se esperaba).

> [!NOTE]
> El valor real de la cadena de criterio de ordenación no es significativo.

#### <a name="enumeration-pair"></a>Par (enumeración)

Las dimensiones que se proporcionan como un par (enumeración) se muestran con el siguiente formato:

 _\<valor de enumeración de un extremo\> : \<valor de la enumeración desde el otro extremo\>_

El orden de los valores de enumeración es coherente, pero no refleja el orden de los extremos de la primeros o segundo.

Por ejemplo, el par de todos los detalles de conexión de red muestra los valores de detalles de la conexión de red para los dos extremos:

|||
|:-----|:-----|
|**Par de todos los detalles de conexión de red** <br/> |**Cómo interpretar** <br/> |
|Con cable: con cable  <br/> |Primeros y segundo extremos ambos usados con cable conexiones ethernet.  <br/> |
|Con cable: wifi  <br/> |Primer extremo utilizado con cable conexión ethernet y la segunda conexión Wi-Fi de extremo que utilizó, o el segundo extremo usa conexión ethernet cableada y primer extremo usa conexión Wi-Fi.  <br/> |
|: wifi  <br/> |Primer extremo usa una conexión WiFi y la conexión de red utilizada por el segundo extremo es desconocida, o el segundo extremo usa una conexión WiFi y la conexión de red utilizada por el primer extremo es desconocida.  <br/> |
|…  <br/> |…  <br/> |

#### <a name="blank-values"></a>Valores en blanco

La tabla anterior enumera posibles motivos de por qué una dimensión puede estar en blanco. Muchas de las dimensiones y medidas estará en blanco si la dimensión QoE registro disponible es false. Normalmente, esto se produce cuando la llamada no se ha establecido correctamente.
