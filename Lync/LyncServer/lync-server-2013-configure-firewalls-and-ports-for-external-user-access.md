# <a name="configure-firewalls-and-ports-for-external-user-access-in-lync-server-2013"></a>Configurar firewalls y puertos para el acceso de usuarios externos en Lync Server 2013

</div>

<div id="mainSection">

<div id="mainBody">

<span> </span>

_**Última modificación del tema:** 2012-05-21_

Para configurar firewalls y puertos, debe configurarlos para servidores perimetrales, servidores proxy inversos y probablemente equilibradores de carga de hardware (para una implementación escalada que no usa equilibrio de carga de DNS). En esta sección se proporciona información sobre los requisitos de firewall y de puerto para todos los componentes de servidores perimetrales y la configuración de los puertos de firewall para servidores perimetrales. Para obtener más información sobre cómo configurar puertos para servidores de proxy inverso, consulte [configuración de servidores proxy inversos para Lync Server 2013](lync-server-2013-setting-up-reverse-proxy-servers.md). Si está implementando una topología perimetral escalada y está usando el equilibrio de carga de hardware en lugar del equilibrio de carga de DNS, consulte [escalado consolidado ampliado con equilibradores de carga de hardware en Lync Server 2013](lync-server-2013-scaled-consolidated-edge-with-hardware-load-balancers.md) en la documentación de planeación para obtener información detallada sobre cómo configurar los puertos para equilibradores de carga de hardware.
