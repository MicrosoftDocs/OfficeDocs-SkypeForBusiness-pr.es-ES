# <a name="documentation-in-lync-server-2013"></a>Documentación en Lync Server 2013

</div>

<div id="mainSection">

<div id="mainBody">

<span> </span>

_**Última modificación del tema:** 2015-05-15_

El modelo de MOF se compone de muchas funciones de administración de servicios. La documentación sobre cómo y cuándo se realizan las tareas se puede compartir con los miembros del mismo equipo o con otros equipos. El método de almacenamiento y uso compartido de la documentación puede variar según el tipo de función. Por ejemplo, los procedimientos para la administración del sistema pueden almacenarse como documentos de Word porque es probable que se impriman y hagan referencia a ellos con frecuencia. La información de administración de configuración se puede generar automáticamente y almacenar en una base de datos para facilitar la búsqueda y la indización. Algunos documentos pueden ser confidenciales y deben restringirse.

<div>

## <a name="document-management-systems"></a>Sistemas de administración de documentos

Un sistema de administración de documentación actúa como un repositorio central para los documentos y ayuda a garantizar que solo esté disponible la última revisión de un documento. También puede considerar la posibilidad de archivar la versión anterior del documento como referencia. Lync Server ofrece una funcionalidad adecuada para esta tarea.

</div>

<div>

## <a name="databases"></a>Databases

Se trataron varias herramientas y funciones de administración que resultan adecuadas para usar bases de datos. Es probable que el proceso de administración de la configuración use procesos automatizados que almacenen grandes cantidades de datos que requieran indización y búsqueda. El personal de soporte técnico puede buscar en una base de datos de problemas y soluciones anteriores al solucionar problemas nuevos.

Es probable que se usen diferentes bases de datos para distintos fines. Decida si estas bases de datos deben vincularse o combinarse. Por ejemplo, si el servicio de asistencia al cliente identifica varios problemas con un tema común (como software nuevo que causa un problema con un adaptador de red determinado), el personal de soporte técnico puede consultar la base de datos de configuración para predecir cuántos equipos podrían verse afectados.
