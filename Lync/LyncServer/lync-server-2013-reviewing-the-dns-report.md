# <a name="reviewing-the-dns-report-in-lync-server-2013"></a>Revisión del informe DNS en Lync Server 2013

<div data-xmlns="http://www.w3.org/1999/xhtml">

<div class="topic" data-xmlns="http://www.w3.org/1999/xhtml" data-msxsl="urn:schemas-microsoft-com:xslt" data-cs="https://msdn.microsoft.com/">

<div data-asp="https://msdn2.microsoft.com/asp">



</div>

<div id="mainSection">

<div id="mainBody">

<span> </span>

_**Última modificación del tema:** 2013-02-21_

El informe de DNS, que forma parte del informe de administrador, detalla todas las entradas recomendadas y conocidas para el sistema de nombres de dominio (DNS) en las redes internas, perimetrales y externas. Si el diseñador ha completado las modificaciones en el diagrama de red y todas las direcciones IP y los nombres de dominio completos (FQDN) se definen en sus valores de producción, el informe de DNS proporciona un recurso de configuración excelente. Este informe también puede servir como un documento de solución de problemas operacional.

![Informe de administración de DNS](images/Gg558641.9dd1e810-ddc7-4816-a806-4239baf9ec51(OCS.15).jpg "Informe de administración de DNS")

Debe hacer que el equipo de administración de DNS Revise el informe de DNS minuciosamente para asegurarse de que no haya errores que puedan causar dificultades durante la implementación o que puedan complicar una sesión de solución de problemas.
