# <a name="manage-hosted-voice-mail-policies-in-lync-server-2013"></a>Administrar directivas de correo de voz hospedado en Lync Server 2013

</div>

<div id="mainSection">

<div id="mainBody">

<span> </span>

_**Última modificación del tema:** 2012-09-20_

Una *Directiva de correo de voz hospedada* proporciona información a la aplicación de enrutamiento de 2013 ExUM de Lync Server acerca de dónde enrutar las llamadas de los usuarios cuyos buzones se encuentran en un servicio de Exchange hospedado.

<div>


> [!NOTE]  
> Por lo general, solo se necesita una directiva de correo de voz hospedado. En muchos casos, puede modificar la directiva global para satisfacer todas sus necesidades. Si crea una directiva con el ámbito del sitio, se asignará automáticamente a todos los usuarios alojados en el sitio especificado. Si crea una directiva con ámbito por usuario, debe asignarla explícitamente a los usuarios, grupos y objetos de contacto. Es posible implementar varias directivas de correo de voz hospedado, pero, en ese caso, las directivas deben asignarse para cada usuario.



</div>

Para obtener más información sobre cómo planear directivas de correo de voz hospedado, consulte [directivas de correo de voz hospedado en Lync Server 2013](lync-server-2013-hosted-voice-mail-policies.md) en la documentación de planeación.
