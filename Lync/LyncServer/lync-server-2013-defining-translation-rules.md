# <a name="defining-translation-rules-in-lync-server-2013"></a>Definición de reglas de conversión en Lync Server 2013

_**Última modificación del tema:** 2013-02-22_

Lync Server 2013 Enterprise Voice enruta las llamadas en función de los números de teléfono normalizados al formato E. 164. Esto significa que todas las cadenas marcadas deben normalizarse al formato E. 164 para realizar búsquedas inversas de números (RNL), de modo que se puedan traducir al URI del SIP correspondiente. Lync Server 2013 proporciona la capacidad de manipular el identificador llamado y la presentación del identificador de llamada.

En esta sección se explica cómo manipular el identificador llamado y el identificador del autor de la llamada.
